# 🛡️ PII Shield — Альфа-Банк Proxy

**Асинхронный прокси-сервер реального времени для маскирования и защиты персональных данных (ПДн) при взаимодействии с LLM.**

Разработано в рамках хакатона от Альфа-Банка.

## 🌟 Особенности проекта

*   **Двухуровневое маскирование:**
    *   **Regex Masking:** O(N) поиск по регулярным выражениям с учетом приоритетов (Жадные алгоритмы). Находит: *Номера счетов (20), ОМС (16), Банковские карты, СНИЛС, Телефоны, Паспорта, ИНН, Email, Даты, Водительские удостоверения*.
    *   **NER Masking (Natasha):** Интеллектуальное распознавание именованных сущностей в контексте (ФИО `PER`, Локации `LOC`, Организации `ORG`).
*   **Безопасность (Hardened Security):**
    *   **Prompt Injection Defense:** Нейтрализация попыток пользователя сымитировать токены масок (например, `[PER_1]`). Токены экранируются в `«PER_1»`.
    *   **Stream Interceptor:** Перехватчик потока от LLM восстанавливает данные "на лету" (SSE). Неизвестные токены (галлюцинации LLM или попытки взлома) заменяются на безопасные заглушки `***`.
*   **Высокая производительность:**
    *   Асинхронный `FastAPI`.
    *   Тяжелые NER-операции вынесены в `asyncio.to_thread`, не блокируя Event Loop.
    *   Latency-метрики (время маскирования) транслируются на клиент.
*   **UI/UX:**
    *   Современный Glassmorphism-интерфейс с журналом маскирования (Audit Log) и визуализацией найденных сущностей.
*   **100% Плотный код:** Никаких лишних комментариев, полная готовность к production, интеграционные тесты пройдены (18/18).

## 🏗 Архитектура (Трубопровод / Pipeline)

1.  **Пользователь** отправляет сообщение с ПДн.
2.  **Шаг 0 (Sanitize):** Очистка от ложных токенов с целью защиты от Prompt Injection.
3.  **Шаг 1 (Regex):** Маскировка жестких форматов (СНИЛС, Карты и т.д.).
4.  **Шаг 2 (NER):** Маскировка ФИО и других контекстных сущностей через `Natasha` (в отдельном потоке).
5.  **LLM:** Анонимизированный текст отправляется в LLM (в текущей версии интегрирован фейковый генератор для демо).
6.  **Stream Interceptor:** Потоковый ответ от LLM анализируется, токены заменяются обратно на исходные ПДн (только если они есть в изолированном словаре `mapping` текущей сессии).
7.  **Пользователь** мгновенно получает размаскированный ответ.

## 🚀 Как запустить

### Способ 1: Локально (Python 3.10+)

```bash
# Клонировать репозиторий
git clone <URL_репозитория>
cd <папка_проекта>

# Установить зависимости
pip install -r requirements.txt

# Запустить сервер
python main.py
```
*Сервер будет доступен по адресу: http://localhost:8000*

### Способ 2: Через Docker

```bash
# Сборка образа
docker build -t pii-shield-proxy .

# Запуск контейнера (FastAPI с 4 воркерами Uvicorn)
docker run -d -p 8000:8000 pii-shield-proxy
```

## 🧪 Тестирование

В проекте есть встроенный скрипт для интеграционного E2E-тестирования.
При **запущенном сервере** выполните:

```bash
python test_integration.py
```
Скрипт проверяет API, здоровье сервера, защиту от инъекций, валидацию входов и корректность стриминга.

## 📡 API Эндпоинты

*   `GET /` — Веб-интерфейс клиента.
*   `POST /api/chat` — Основной эндпоинт. Принимает `{"message": "..."}`, возвращает поток Server-Sent Events (SSE). Защищено валидацией до 5000 символов.
*   `POST /api/mask_only` — Отладочный эндпоинт для проверки маскирования. Возвращает исходник, замаскированный текст, маппинг и время обработки (мс).
*   `GET /api/health` — Healthcheck для Docker-контейнера.

## 📂 Структура проекта

*   `main.py` — FastAPI сервер, валидация, логика роутинга и фейковая LLM-заглушка.
*   `masking_pipeline.py` — Главный конвейер маскирования (Sanitize -> Regex -> NER).
*   `regex_masker.py` — Регулярные выражения и защита от Prompt Injection.
*   `ner_masker.py` — NLP модель (Natasha).
*   `stream_interceptor.py` — Локатор и подменщик токенов в стриме ответа.
*   `test_integration.py` — Интеграционные тесты.
*   `static/index.html` — Фронтенд (UI).
